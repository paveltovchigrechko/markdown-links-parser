The code editor allows you to browse and manage the files used in a store website theme. 

:::caution

The code editor is a powerful tool that allows you to change a theme completely. However, with this power comes responsibility. When you use the code editor to change a theme, you take full responsibility for any consequences that your changes can bring. 

Unistore **does not** take any responsibility for any harm, damage, or other negative impacts to your stores, accounts, customers, and website visitors that changes made in the theme can cause.

:::

By default, the code editor contains files of the active theme (you can check your active theme in [**Theme library**](../themes/themes.mdx)). However, if you use **Edit theme** option in **Theme library**, the code editor contains files of the selected theme.

Note that generally, any changes made in the code of the active theme are applied instantly to your store website. We recommend you work with an inactive theme code, make small changes at the time, and check the impact of your changes in the website preview of the **Page editor**. Potentially dangerous changes are editing the source code, renaming and deleting files.

## Create file {#create-file}

All theme files are distributed into theme directories. Depending on the directory, each file has a specific type and role in a theme, so it is important to create new file in the appropriate directory. <!-- To learn more about theme directories, refer to dev documentation link -->

You can create a theme file in any directory except **Config**.

:::note

All file names must be unique within a directory.

:::

To create a code file:

## Rename file {#rename-file}

:::info

The main layout file `theme.liquid` and all files in **Config** directory are crucial for rendering any theme, so you cannot rename any of these files.

:::

To rename a file:

## Delete file {#delete-file}

:::caution

You cannot restore a deleted file. For that reason, before you delete any file, make sure it is not used anymore in the theme. To do so, you can:

1. Save the file code locally or into another temporary file in the code editor
2. Delete the file
3. Check out your store website in the website preview of the **Page editor**

If your store website misses some functionality or demonstrates some unexpected behavior after deleting a file, you can use the saved source code to restore the theme.
